Using Ocp-TreeNode

Call the Tree via javascript:

KISSY.use('ocp/treeNode', function(S, TreeNode){

	var node= new TreeNode({
		prefixCls : 'ocp-',
		container : '#treeContainer',
		store :store,
		checkable : true
	});
});

Options

Name type default description
content string true 树节点文本内容
isLeaf boolean true 是否为叶子节点，决定该节点是否可展开
checkable boolean false 树节点是否可选，默认不可选
prefixCls string false 样式前缀，默认“ocp-”
id string false 节点的唯一标示ID，默认随机生成
Heads up! If you want your modal to animate in and out, just add a .fade class to the .modal element (refer to the demo to see this in action) and include bootstrap-transition.js.

Propertys

OCP Tree class exposes a few propertys.

Event Description
id 树节点ID，只读
depth 树节点的深度，只读
children 子节点，只读，node.children的形式调用
visible 树节点可见状态，可设置
content 树节点的显示内容，可以包含html片段
dataContent 树节点的纯文本内容
checkState 树节点当前选中状态，false: 未选中、true: 已选中

Methods

.addChild(node. needRender)

新增节点对象，建立父子关系. node 插入的节点对象，needRender 是否需要立即渲染DOM默认false.

.toggleCheck()

节点选中状态切换.

.toggleExpand()

节点收起展开状态切换

.expand()

节点展开

.collapse()

节点收起

.findNode(attr, value, fn)

查找符合匹配条件的子节点. attr 查询的节点属性.value 匹配值.fn 匹配时所做的操作，参数为节点对象node.

.cascade(fn, scope)

中序遍历遍历所有子节点. fn遍历函数 返回值为false则继续循环遍历，True则终止循环，参数为当前节点对象

.destroy()

摧毁节点的所有DOM引用以及对象引用，并摧毁子节点

Using Ocp-Tree

Call the Tree via javascript:

KISSY.use('ocp/tree, ocp/store', function(S, Tree, Store){
	var store = new Store({
		//"url" : 'treeJosn.html',
		"data" : [
			{
				"content" : '属性一',
				"isLeaf" : false,
				"children" : [
					{
						"content" : '属性值一 一'
					},
					{
						"content" : '属性值一 二',
						"isLeaf" : false,
						"children" : [
							{
								"content" : '属性值'
							},
							{
								"content" : '属性值'
							}
						]
					}
				]
			},
			{
				"content" : '属性值二',
				"isLeaf" : false
			}
		]
	});
	var tree = new Tree({
		prefixCls : 'ocp-',
		container : '#treeContainer',
		store :store,
		checkable : true
	});
	tree.render();
	//store.load();
});

Json Data Format

Name type default description
content string true 节点的文本内容
isLeaf boolean true 节点是否叶子节点，绝对该节点是否可展开
children node json data true 子节点JSON数组
checkable boolean false 节点是否可选，默认false 不可选
id string false 节点ID，可以不提供，默认随机生成
Heads up! 构造树的Json数据格式，除了以上关键属性以外，可以任意添加其他属性，会自动mixin Node对象中，作为扩展属性使用

Options

Name type default description
container html el or selector true 需要渲染树控件的容器DOM节点
store ocp store true 数据模型对象，包含树的初始数据或者可异步访问的URL
data json object true 树的初始数据，和Store二选其一
beforeRender function false 每个树节点渲染之前调用，参数为节点内容和节点对象
checkable boolean false 树是否可勾选，默认为false
prefixCls string false 样式前缀，默认为ocp-
id string false 树的唯一标示ID，默认随机生成

Methods

.render()

当一棵树内部数据，或所有节点关系更新以后，将树渲染到DOM中

tree.render()

.findNode(attr, value)

根据相应属性匹配规则,查找节点.attr:树节点的属性;value:属性的值;

返回查找到的树节点对象

tree.findNode('id', '29803')

.filter(fn)

根据相应匹配规则过滤节点，隐藏过滤掉的节点. fn:遍历每个节点时调用回调函数，参数为节点对象，返回truefalse来设置该节点是否可见

tree.filter(function(node){
	node.set('conent', 'matched!')
	return true;
})

.destroy()

消耗树中所有节点

.getCheckedNodes()

取得所有被勾选的节点.

tree.getCheckedNodes()

.getSelectedNode()

取得被选中的节点.

tree.getSelectedNode()

Forest ocp/forest

About Forest

森林控件，包含多个树，并具备左右轮播的动画效果.

Example Demo

Below is a statically rendered modal.

Using Ocp-Tree

Call the Tree via javascript:

KISSY.use('ocp/forest, ocp/store', function(S, Forest, Store){
	var store = new Store({
		"url" : 'firstCategoryJson.html'
	});
	forest = new Forest({
		prefixCls : 'ocp-',
		width : 250,
		margin : 4,
		container : '#forestContent',
		store :catStore
	});
	catStore.load();
	
	forest.on('click', function(e){
		var id = e.treeId,
			node = e.node,
			// 取得节点的附加数据
			cpv = node.get('urlParam'),
			isleaf = node.get('isleafCat');

		// 先销毁后面的树
		this.destroyAllTree(id);
		this.createNewTree(catStore, true);
		catStore.load();
	});
});

Options

Name type default description
container html el or selector true 需要渲染森林控件的容器DOM节点
store ocp store true 数据模型对象，包含森林中第一颗树的初始数据或者可异步访问的URL
width number true 单颗树的宽度
height number true 单颗树的高度
treeNum number false 森林可视区域显示的树个数，默认3
prefixCls string false 样式前缀，默认为ocp-
placeholder string false 搜索输入框的placeholder 默认（输入关键字）
Heads up! If you want your modal to animate in and out, just add a .fade class to the .modal element (refer to the demo to see this in action) and include bootstrap-transition.js.

Methods

.createNewTree(store, checkable)

创建新树对象，追加至森林结尾。store树的数据模型对象，checkable树是否可选，默认不可选

.destroyTree(index)

摧毁森林中指定的树对象.index: 森林中树序列的数组序号;

.destroyAllTree(startId)

毁指定索引开始之后的所有树对象. startId:树在森林中的唯一标示符

.getTrees()

取得森林中的树对象数组

Events

	var node = e.node,
		id = e.treeId;

Using S.Dialogbox.box

通过S.Dialogbox声明一个dialogbox实例,建议使用box方法，多次使用不会多次声明实例

S.Dialogbox.box(msg,callback,options)
new S.Dialogbox.(options) //裸声明

options

Name type default description
clickNode string 空 绑定Dom节点的click事件，支持selector绑定
srcNode string 空 用于取悬浮层根节点, 可为”#id”或”.class”. 当不设置时表示新建一个 HTMLElement 插入到页面中.初始 srcNode 不能设置为 display:none ，需要设置为 position:absolute;left:-9999px;top:-9999px
head string 空 头部html内容设置
body string 空 主体内容html设置
foot string 空 底部内容html设置
mask boolean true 是否带有遮罩
onload function 不执行 初始化完成后的回调，在render后立即执行
resize Object null 开启可缩放并设置可缩放参数
draggable boolean true 是否可拖拽
prefixCls function string 默认为”ks-”, 样式类名前缀, 如悬浮层根元素会加上样式类：”ks-overlay”. kissy 1.2 版本以前设置无效, 都为 “ks-”
